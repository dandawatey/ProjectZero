# Workflow: Incident Response

## Steps
1. **Detect**: Alert fires from monitoring, user report, or automated check
2. **Triage**: Assign severity (P1-P4), assign incident commander
3. **Communicate**: Update status page, notify stakeholders
4. **Investigate**: Check logs, metrics, traces. Identify root cause.
5. **Mitigate**: Apply immediate fix (rollback, feature flag, hotfix)
6. **Resolve**: Permanent fix with tests, through expedited governance chain
7. **Postmortem**: Blameless review within 48 hours
8. **Prevent**: Create tickets for preventive measures

## Severity Response Times
- P1 (Critical): 15 minutes
- P2 (High): 1 hour
- P3 (Medium): 4 hours
- P4 (Low): Next business day
