# Workflow: SPARC: Specification

## Purpose
Specification phase of SPARC

## Entry Criteria
Approved BMAD, module candidates

## Process
Product Manager agent extracts requirements, creates specifications with acceptance criteria, creates epics and stories, prioritizes backlog, validates through governance chain

## Governance
All artifacts pass through maker-checker-reviewer-approver chain.

## Exit Criteria
All deliverables complete, validated, and approved. Ready for next phase.

## Failure Handling
- Checkpoint state before each step
- On failure: log to failure-log.md, attempt recovery, escalate after 3 retries
- Never skip steps or bypass governance
