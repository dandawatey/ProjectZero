# UX Rules

1. Every user action needs visible feedback (button press, form submit, navigation)
2. Loading states required for any async operation (spinner, skeleton, or progress)
3. Error states must explain what went wrong and how to fix it
4. Empty states guide user toward first action (not just "No data")
5. Forms validate inline (on blur), not just on submit
6. Navigation is consistent across all pages
7. Destructive actions require confirmation
8. Undo is preferred over confirmation where possible
9. Keyboard shortcuts for power users (documented, discoverable)
10. Mobile touch targets minimum 44x44px
