# Skill: Playwright E2E Testing

## Purpose
End-to-end testing with Playwright for critical user flows.

## Methodology
1. Identify critical flows: Login, core feature, payment, etc. 2. Create page objects: Encapsulate page interactions. 3. Write test scenarios: Happy path, error path, edge cases. 4. Use stable selectors: data-testid attributes preferred. 5. Add assertions: Verify visible outcomes, not implementation. 6. Configure CI: Run in headless mode, generate reports.

## Stage Mapping
Used during: Realization, Completion (unless otherwise noted)

## Integration
Available to all agents. Invoke when trigger conditions are met.
