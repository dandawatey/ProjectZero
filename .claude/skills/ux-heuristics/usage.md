# Usage: UX Heuristics

## How to Invoke
Activate when trigger conditions are met (see triggers.md).

## Process
Evaluate each heuristic (score 0-4, 0=no issue, 4=catastrophic):
1. Visibility of system status: Does user know what's happening?
2. Match real world: Does it use user's language and concepts?
3. User control and freedom: Can user undo/redo, navigate freely?
4. Consistency: Same actions, same results everywhere?
5. Error prevention: Does design prevent errors before they happen?
6. Recognition over recall: Is information visible, not memorized?
7. Flexibility: Shortcuts for experts, simplicity for novices?
8. Aesthetic and minimalist: No unnecessary information?
9. Error recovery: Are error messages helpful with clear resolution?
10. Help and documentation: Is help available when needed?
