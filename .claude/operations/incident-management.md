# Incident Management

## Process
1. **Detect**: Monitoring alert, user report, or automated check
2. **Alert**: Page on-call (P1/P2), create incident channel
3. **Triage**: Assign severity, assign incident commander
4. **Investigate**: Logs, metrics, traces — identify root cause
5. **Mitigate**: Rollback, feature flag, hotfix — stop the bleeding
6. **Resolve**: Permanent fix through expedited (but governed) process
7. **Postmortem**: Blameless review within 48 hours
8. **Prevent**: Create tickets for preventive measures

## Postmortem Template
1. Summary (1 paragraph)
2. Timeline (minute-by-minute for P1/P2)
3. Impact (users affected, duration, data impact)
4. Root Cause (technical details)
5. Resolution (what fixed it)
6. Action Items (with owners and due dates)
7. Lessons Learned
