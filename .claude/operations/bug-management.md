# Bug Management

## Workflow
Report → Triage → Assign → Fix → Verify → Close

## Triage Rules
- **P1 (Critical)**: Service down, data loss, security breach. SLA: 4 hours to fix.
- **P2 (High)**: Major feature broken, no workaround. SLA: 1 business day.
- **P3 (Medium)**: Feature broken, workaround available. SLA: 1 sprint.
- **P4 (Low)**: Cosmetic, minor inconvenience. SLA: backlog (prioritize as capacity allows).

## Bug Ticket Requirements
- Steps to reproduce (exact, numbered)
- Expected behavior
- Actual behavior
- Environment (browser, OS, version)
- Screenshots/logs if available
- Severity assessment
