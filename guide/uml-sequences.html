Diagram 2 — MCRA 4-Eye Approval

MCRA 4-Eye Approval — Maker → Checker → Reviewer → Approver
sequenceDiagram autonumber participant Dev as Developer participant Tmp as Temporal participant Ch as Checker participant Rev as Reviewer participant Ap as Approver participant JIRA as JIRA participant Br as Brain Dev->>Tmp: POST /tickets/{id}/approve — submit for MCRA review Tmp->>Ch: mcra_checker_activity Note over Ch: run quality gates
pytest · ruff · pyright Ch-->>Tmp: GateResult(passed/failed, details) alt quality gates failed Tmp->>JIRA: add comment "Checker FAILED: {details}" Tmp-->>Dev: MCRAResult(status=failed) end Tmp->>Rev: mcra_reviewer_activity Rev->>Br: GET /brain/patterns + /brain/decisions Br-->>Rev: prior patterns + decisions Rev->>Rev: call Claude API → review artifact Rev->>JIRA: add comment "Reviewer: {verdict}" Rev-->>Tmp: verdict(approved / changes_required) alt reviewer requests changes Tmp-->>Dev: MCRAResult(status=changes_required) end Note over Tmp: wait for human APPROVER signal
72h timeout Tmp->>JIRA: add comment "Awaiting APPROVER" Dev->>Tmp: signal: mcra_approve(approved=true, comment) Tmp->>JIRA: add comment "APPROVED by {approver}" Tmp->>Br: POST /brain/decisions — write decision memory Tmp-->>Dev: MCRAResult(status=approved, pr_url)
