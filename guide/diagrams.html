9. Brain (Persistent Memory Architecture)

Postgres-backed Brain — /api/v1/brain/

Memories

  • Scoped: factory / product / session
  • Categorized + tagged
  • Promotable upward
  • Usage tracking

Decisions

  • Context + options
  • Chosen + rationale
  • Supersedable
  • Full audit trail

Patterns

  • Problem → Solution
  • Anti-patterns
  • Success rate tracked
  • Cross-product proven

Conversations

  • Per workflow step
  • 4 modes: chat / brainstorm / plan / implement
  • Decisions extracted
  • Persists across sessions
Agents read Brain before action → execute → write Brain after action. Memory promotes: session → product → factory.

13. Recovery Architecture

Temporal Never Loses State

IDE Crash

  • Temporal continues running
  • /resume reloads context
  • Brain preserves conversation
  • Zero data loss

Workflow Failed

  • Auto-retry (max 3)
  • Exponential backoff
  • If still fails → escalate
  • Human decides via UI

Context Overflow

  • Checkpoint to Brain
  • Start new session
  • /resume loads from Brain
  • Continue from checkpoint

14. Repo Boundary Model

Factory Repo vs Product Repo — Strict Separation

ProjectZeroFactory (Factory)

  • 35 agent definitions
  • 17 skill packages
  • 21 workflows
  • 25 commands
  • Templates + product skeleton
  • platform/ (FastAPI + React + Temporal)
  • Governance + guardrails
  • Brain (Postgres — shared)

NEVER contains product code or state

Product Repo (Per Product)

  • src/ — application code
  • tests/ — test suites
  • packages/ — monorepo packages
  • .claude/delivery/ — JIRA/Confluence state
  • .claude/memory/ — product memory
  • .claude/recovery/ — checkpoints
  • .env — product secrets
  • docs/ — product documentation
