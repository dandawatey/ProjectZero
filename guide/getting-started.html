Getting Started

Go from zero to a managed AI product in three commands. This guide walks you through cloning, initializing, and bootstrapping your first project.

Prerequisites

Before you begin, make sure you have the following installed:

  • Docker — for Postgres and Temporal containers
  • Node.js 18+ — for the React Control Tower
  • Python 3.11+ — for the FastAPI backend
  • Claude Code CLI — the agent runtime
Tip

If you are on macOS, brew install docker node python gets you most of the way. Claude Code CLI is installed separately via npm install -g @anthropic-ai/claude-code.

Step-by-Step Setup

  1. Clone the repository 
    git clone https://github.com/your-org/ProjectZeroFactory.git
cd ProjectZeroFactory
  2. Initialize the factory

    Run the factory initialization command inside Claude Code:

    /factory-init

    This will:

    • Spin up Postgres and Temporal via Docker Compose
    • Run database migrations
    • Create the .factory/ configuration directory
    • Validate your environment and API keys
  3. Bootstrap your first product 
    /bootstrap-product

    You will be prompted for a product name. This creates the product workspace, registers it in the database, and prepares the phase pipeline.

    Important

    Product names must be lowercase, alphanumeric, and may contain hyphens. Example: my-saas-app

Choose Your Path

After bootstrapping, your next step depends on where you are in the product lifecycle:

Have a PRD already?
Jump straight into specification generation. Your PRD will be parsed, validated, and decomposed into stories.
/spec
Just have a vision?
Start from a high-level idea. The agent will interview you, research the market, and produce a complete PRD.
/vision-to-prd
Tip

You can always run /check at any point to see the current state of your product, which phase it is in, and what the next available actions are.

What Happens Next

Once you choose your path, ProjectZero takes over with governed automation:

  1. Agents execute the workflow Specialized agents perform research, write specs, generate architecture, and produce code — all within Temporal workflows that guarantee completion.
  2. Approval gates pause for humans At every critical decision point, the workflow pauses and waits for your explicit approval via /approve. Nothing progresses without your sign-off.
  3. Artifacts are versioned Every PRD, spec, architecture doc, and code file is stored in version-controlled product directories with full audit trails.
  4. Monitor everything Open the Control Tower (/monitor) to watch agent activity, workflow state, and approval queues in real time.

Example: Full Flow

# Initialize once per machine
/factory-init

# Create a new product
/bootstrap-product
  Product name: my-saas-app
  Created product workspace at ./products/my-saas-app/

# Start from vision
/vision-to-prd
  Vision: "A platform for teams to share and review design assets"
  Starting business discovery workflow...
  [Agent] Researching market landscape...
  [Agent] Generating PRD draft...
  [Gate] PRD ready for review. Run /approve to continue.

# Review and approve
/approve
  PRD approved. Moving to Phase 3: Specification.
