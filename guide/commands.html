Command Reference

All slash commands available in ProjectZero, organized by scope. Each command maps to a Temporal workflow.

Factory Commands

CommandPhaseScopePurposeTemporal Workflow
/factory-init Setup Factory Initialize a new ProjectZero factory instance with core configuration, agent registry, and workflow templates. FactoryInitWorkflow
/bootstrap-product Setup Factory Bootstrap a new product within the factory—creates repo, ticket board, environments, and initial configuration. BootstrapProductWorkflow
/factory-audit Any Factory Audit factory health: agent availability, workflow status, configuration integrity, and resource usage. FactoryAuditWorkflow
/factory-upgrade Any Factory Upgrade factory components: agent definitions, workflow templates, governance rules, and platform dependencies. FactoryUpgradeWorkflow

Product Commands

CommandPhaseScopePurposeTemporal Workflow
/vision-to-prd Vision Product Transform a product vision statement into a detailed Product Requirements Document with market analysis. VisionToPRDWorkflow
/business-docs Vision / Planning Product Generate business documents: TAM-SAM-SOM, competitive analysis, financials, GTM strategy, pitch deck. BusinessDocsWorkflow
/setup Setup Product Configure product development environment: tech stack, repo structure, CI/CD, and development tooling. SetupWorkflow
/spec Specification Product Create detailed technical specification from PRD requirements, including acceptance criteria and edge cases. FeatureDevelopmentWorkflow
/arch Architecture Product Design system architecture: component diagram, data models, API contracts, technology decisions. FeatureDevelopmentWorkflow
/implement Implementation Product Execute implementation of a feature or ticket according to spec and architecture. Creates code and tests. FeatureDevelopmentWorkflow
/check Testing Product Run QA validation: execute test suites, check coverage thresholds, validate against acceptance criteria. QAValidationWorkflow
/review Review Product Trigger code review with maker-checker-reviewer pattern. Reviews code quality, security, and standards. MakerCheckerReviewerWorkflow
/approve Approval Product Submit for stakeholder approval. Validates all quality gates are passed before allowing progression. ReleaseGovernanceWorkflow
/release Release Product Execute release pipeline: deployment readiness checks, staging deployment, production promotion. DeploymentReadinessWorkflow
/monitor Post-Release Product Monitor production deployment: health checks, error rates, performance metrics, and rollback triggers. MonitoringWorkflow
/optimize Post-Release Product Analyze performance data and suggest optimizations for code, infrastructure, and user experience. OptimizationWorkflow

Recovery Commands

CommandPhaseScopePurposeTemporal Workflow
/resume Any Recovery Resume a paused or interrupted workflow from its last checkpoint. Restores full workflow context. ResumeWorkflow
/recover-ticket Any Recovery Recover a stuck ticket: diagnose failure point, restore state, and re-queue for processing. TicketRecoveryWorkflow
/recover-workflow Any Recovery Recover a failed Temporal workflow: inspect history, identify failure, attempt repair or restart. WorkflowRecoveryWorkflow

UI Commands

CommandPhaseScopePurposeTemporal Workflow
/design-system-init Setup UI Initialize the product design system: tokens, color palette, typography, spacing, and base components. DesignSystemWorkflow
/component-create Implementation UI Create a new UI component with props, variants, accessibility, tests, and Storybook documentation. ComponentWorkflow
/component-review Review UI Review a UI component for design system compliance, accessibility, responsiveness, and visual quality. ComponentReviewWorkflow
/story-create Implementation UI Create Storybook stories for a component: default state, variants, edge cases, and interactive demos. StoryWorkflow
/story-validate Testing UI Validate Storybook stories: visual regression checks, interaction tests, and accessibility audit. StoryValidationWorkflow
/ui-audit Any UI Audit the full UI layer: design system adherence, component usage, accessibility compliance, and consistency. UIAuditWorkflow
