Phase Lifecycle

Every product in ProjectZero progresses through a governed, sequential pipeline of phases. Each phase is backed by a Temporal workflow that orchestrates agent work, produces artifacts, and gates on human approval before advancing.

Phase Transitions

Every phase transition follows the same governance pattern:

  1. Workflow completes — all activities in the phase finish and artifacts are produced
  2. Artifacts persisted — sync_state() writes results to Postgres
  3. Approval gate — workflow pauses on a Temporal signal-wait
  4. Human review — user reviews artifacts in Control Tower or CLI
  5. Approval signal — /approve sends the signal, workflow resumes
  6. Next phase starts — the subsequent workflow is automatically triggered

If a phase needs to be re-run (e.g., after feedback), use /recover-workflow to reset and restart it. The previous artifacts are preserved in version history.
